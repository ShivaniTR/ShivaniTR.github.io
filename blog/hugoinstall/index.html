Installing Hugo, Trial and Error

Jun 7, 2021 | 2 minutes read
Share this: Twitter Facebook Pinterest

Tags: hugo, install, uninstall

It took me three attempts to successfully install Hugo. I realised the consequence of not reading the doccumentation and instructions properly.

First Attempt

I used the apt-get command line tool to install hugo.

sudo apt-get update
sudo apt-get install hugo

I went on to explore the themes and and I came across this: *Minimum Hugo Version: 0.82.0

On checking:

hugo version 
Hugo Static Site Generator v0.40.1 linux/amd64 BuildDate:2018-04-25T17:16:11Z

I found that this is the default version the apt-get tool installs

Second Attempt

So I found THIS site and found a way to choose which version I want to install. I checked the Hugo release page and did the folowing:

: wget https://github.com/gohugoio/hugo/releases/download/v0.83.1/hugo_0.83.1_Linux-64bit.deb 
: sudo dpkg -i hugo_0.83.1_Linux-64bit.deb
: sudo rm hugo_0.83.1_Linux-64bit.deb 
: sudo hugo version 
:Hugo Static Site Generator v0.83.1 linux/amd64 BuildDate:2018-09-25T17:16:11Z

After this I proceeded to install a theme and adding content to my site.

When I tried to host it locally:

hugo server -D

It showed me an error message that the extended version of Hugo is required.

Third Attempt

Finally, I went back to the hugo site and looked at their [doccumentation] (https://gohugo.io/getting-started/installing/). They reccomended using homebrew to install Hugo.

So I installed homebrew with the steps given HERE Then I proceeded to do:

brew install hugo

And I finally got the correct extended version.

Terminal recording of final installation.

I wasted quite a lot of time over this and I will always read the doccumentation and instructions carefully from now.
